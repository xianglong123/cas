隔离级别分四种：
1. 未提交读：优势 -> 如果对数据的一致性没那么高，支持高并发场景  劣势 -> 造成 脏读，不可重复读，幻读
2. 读写提交：优势 -> 克服脏读 劣势 -> 牺牲一部分性能克服脏读，但还是会造成不可重复读，幻读
3. 可重复读：优势 -> 克服脏读，不可重复读 劣势 -> 理解为行锁，牺牲性能，会造成幻读
4. 串行化：  优势 -> 克服脏读，不可重复读，幻读 劣势 -> 理解为多线程变成单线程，它将sql排序执行，单线程就不存在脏读，幻读，不可重复读。但是会造成性能降到最低

oracle 支持 读写提交和串行化 默认隔离级别：读写提交

mysql 支持 未提交读，读写提交，可重复读，串行化  默认隔离级别：可重复读
